OPERATING TOOL · RECORD 01

Correction Qualification Record

A practical candidate-created record for deciding whether a correction remains shadow-only, may proceed inside a bounded release, requires human confirmation, or must hold.

CORRECTION CLASS

Thermal drift / tool wear / fixture shift / program ambiguity / other

RECORD ID + VERSION

Machine · operation · timestamp · model/rule version

MANUFACTURING INTENT

Part/feature, coordinate frames, program/CAM reference, tolerance, expected geometry, operator intent.

OBSERVED RESIDUAL

Measured deviation, sensing/metrology method, units, repeatability, uncertainty.

STATE RECONSTRUCTION

Machine, tool, part, fixture, material, environment, telemetry, missing context.

CAUSE HYPOTHESIS

Measured facts, inferred cause, alternative causes, confidence, falsification test.

CORRECTION LOCUS

Coordinate transform, offset, program/CAM, process parameter, maintenance action, hold.

AUTHORITY + LIMITS

Recommendation / human approval / bounded auto-correction; magnitude, rate, duration, scope, hold and rollback conditions.

OPERATING TOOL · RECORD 02

Proof, disposition, and learning admission

PRE-ACTION PROOF

Replay, simulation, collision-safe check, shadow inference, comparable history.

PERMITTED ACTION

Exact intervention, responsible actor, release window, rollback owner.

POST-ACTION VALIDATION

Metrology result, residual before/after, repeatability, unintended effect, fallback used.

INCIDENT / EXCEPTION

Unexpected state, boundary violation, disagreement, model/rule failure, operator observation.

REUSE BOUNDARY

Machine, controller, tool, material, operation, program style, environment, fixture/setup, version and invalidation triggers.

LEARNING ADMISSION DECISION

Retain as local evidence / admit to qualified class / continue shadow / reject and investigate. Name the approver and supporting evidence.

Final disposition
BOUNDED CORRECTION
QUALIFIED ADJUSTMENT
CONFIRM REGISTRATION
HUMAN HOLD REQUIRED
A correction is not reusable because it worked once.

It becomes reusable only when the state, evidence, authority, validation, and scope that made it work remain explicit and versioned.
